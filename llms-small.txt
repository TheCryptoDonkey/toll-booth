# toll-booth

HTTP 402 payment middleware for Node.js. Gates any API behind Lightning, Cashu, NWC, x402, or IETF Payment (draft-ryan-httpauth-payment-01) - the only middleware supporting all five rails simultaneously.

## Install

```bash
npm install @forgesworn/toll-booth
```

## Minimal Example

```typescript
import express from 'express'
import { Booth } from '@forgesworn/toll-booth'
import { phoenixdBackend } from '@forgesworn/toll-booth/backends/phoenixd'

const app = express()
const booth = new Booth({
  adapter: 'express',
  backend: phoenixdBackend({ url: 'http://localhost:9740', password: process.env.PHOENIXD_PASSWORD! }),
  pricing: { '/api': 10 },
  upstream: 'http://localhost:8080',
})

app.get('/invoice-status/:paymentHash', booth.invoiceStatusHandler as any)
app.post('/create-invoice', booth.createInvoiceHandler as any)
app.use('/', booth.middleware as any)
app.listen(3000)
```

Scaffold a project: `npx @forgesworn/toll-booth init`

## Key Facts

- Multi-rail: L402, x402 stablecoins, Cashu ecash, IETF Payment - all simultaneously
- Adapters: `'express'`, `'web-standard'` (Deno/Bun/Workers); Hono: use `createHonoTollBooth()` from `@forgesworn/toll-booth/hono`
- Backends: Phoenixd, LND, CLN, LNbits, NWC; or Cashu-only (no Lightning node)
- Self-hosted: operators run their own infrastructure, receive their own payments
- Credit system with volume discount tiers
- Optional per-IP daily free tier
- SQLite persistence (WAL mode) or in-memory storage
- Privacy by design; IP addresses are one-way hashed, no PII stored
